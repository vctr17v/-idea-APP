# Reglas del Proyecto

## Codigo generado con IA
- Todo codigo de IA lleva comentario: `// Generated by [tool] - reviewed by [nombre]`
- Todo codigo de IA requiere revision humana antes de merge
- Los tests generados necesitan edge cases manuales

## Checklist de seguridad para codigo IA
Antes de hacer commit, verifica:
- [ ] Sin secretos hardcodeados
- [ ] Inputs validados
- [ ] Error handling explicito
- [ ] Entiendo lo que hace cada linea

## Git workflow
- Ramas: `feature/`, `fix/`, `docs/`
- Commits: `feat(scope): descripcion`
- PRs: requieren 1 revision humana
